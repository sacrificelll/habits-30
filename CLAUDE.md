# CLAUDE.md — правила работы над Habits 30

## Что это за проект

**Habits 30** — простой веб-трекер привычек на 30 дней (до 5 привычек, ежедневная отметка в один клик, календарь прогресса, streak и статистика). Без регистрации и бэкенда — данные в `localStorage`. Деплой на Vercel.

Это **тестовое задание от OPERIX** (сроки 18–22 июня 2026, некоммерческое). Подробности — в `.business/INDEX.md`.

## Структура репозитория

```
./
├── CLAUDE.md                ← этот файл, правила работы
├── README.md                ← документация проекта (сдаётся OPERIX)
├── .business/               ← контекст проекта («второй мозг»)
│   ├── INDEX.md             ← входная точка
│   ├── products/overview.md ← что строим (8 обязательных требований)
│   ├── goals/kpi.md         ← по чему оценивают (критерии OPERIX)
│   └── assets/brand-guidelines.md ← дизайн: палитра, шрифты, темы, UI
├── plans/                   ← планы по фичам (план задания: 2026-06-18-habits-30.md)
├── retrospectives/          ← рефлексии после каждой сессии
└── prompts/                 ← библиотека готовых промптов
```

## Стек

Next.js (App Router) + TypeScript + Tailwind CSS. Хранение — `localStorage` (без бэкенда). Деплой — Vercel.

## Ключевые принципы работы

1. **Перед задачей по продукту/дизайну читай `.business/`** — особенно `products/overview.md`, `goals/kpi.md`, `assets/brand-guidelines.md`. Не выдумывай требования.
2. **ТЗ — закон.** Сначала все 8 обязательных требований, потом доп-фичи. Каждую доп-фичу нужно уметь обосновать одним предложением (см. `.business/execution/backlog.md`).
3. **Один чат — одна задача.** Разные задачи = разные чаты.
4. **План для каждой фичи** — в `plans/YYYY-MM-DD-название.md`: фазы со статусами `[ ]`/`[x]`, итоговый блок. Актуализируй план после сессии.
5. **Ретроспектива в конце сессии** — в `retrospectives/YYYY-MM-DD_HH-MM_название.md`.
6. **Перед push** — запусти билд (`next build`) и линтер. Чини ошибки до пуша.
7. **Чистота кода — критерий оценки.** Без мусора и закомментированных кусков. Осмысленные коммиты (по ним судят о работе с Claude).
8. **Думай за пользователя.** Обрабатывай края: пустое состояние, удаление, сброс, лимит 5, мобильный экран.

## Безопасность

- Не скармливай Клоду пароли, ключи, персональные данные.
- `.env` (если появится) не читай целиком — только нужную переменную через `grep`.
- Перед установкой скила / MCP — аудит через `prompts/setup/08-skills-install.md`.
